Donjon FI — Compagnon VS Code

Aperçu, visualisation, analyse et test du jeu directement dans VS Code.

Démarrage
  1. Ouvrir un fichier .djn dans VS Code.
  2. Lancer le compagnon via Ctrl+F5, ou via l'une des deux icônes / dans la barre de titre de l'éditeur, ou via la palette de commandes (« Ouvrir le compagnon Donjon FI » ou « Tester le jeu Donjon FI »).
  3. Le scénario est compilé une première fois automatiquement.
  4. Après chaque modification du .djn, cliquer sur Analyser pour relancer la compilation. La sauvegarde du fichier ne suffit pas — la recompilation est manuelle.
Onglets
  • Analyse — liste des erreurs, problèmes et conseils. Le badge affiche le nombre total de messages, coloré selon la sévérité maximale (rouge / orange / bleu). Cliquer sur le badge d'un message ouvre le fichier source à la bonne ligne dans VS Code.
  • Jeu — lecteur en direct pour tester le scénario. Sélectionné automatiquement après une compilation sans erreur.
  • Visualisation — 3 sous-onglets pour explorer le monde compilé :
    • Carte — carte ASCII des lieux, objets et personnes ; clic sur un élément pour voir son détail.
    • Visualisation — fiche détaillée d'un lieu (objets, classes, règles, actions associées).
    • Aperçu — vue d'ensemble du monde compilé.
  • Wiki — documentation du DSL Donjon FI intégrée (chargée depuis donjon.fi/doc).
Boutons
  • Analyser — (re)compile le scénario à partir du fichier sur disque. Sélectionne automatiquement l'onglet Analyse en cas d'erreur, sinon Jeu.
  • Télécharger mon jeu — produit un fichier HTML autonome contenant le lecteur Donjon FI + le scénario (multi-fichiers résolus) + les actions, distribuable et jouable hors ligne.
actions.djn — résolution en 3 niveaux
  1. Un actions.djn placé à côté du .djn édité (priorité maximale).
  2. Un actions.djn partagé via le paramètre donjon.actionsFile (chemin absolu ou relatif au workspace).
  3. La version par défaut livrée avec l'extension (toujours à jour avec la dernière version stable).
Inclusion de fichiers

La directive inclure "fichier.djn" permet de découper un gros scénario. Le compagnon résout récursivement tous les inclure avant compilation. Les erreurs sont remontées sur la ligne du bon fichier source. Astuce : Ctrl+Clic sur le nom de fichier dans la directive inclure ouvre le fichier dans VS Code (extension Donjon FI Language requise).

Paramètres VS Code
  • donjon.actionsFile — chemin vers un actions.djn partagé.
  • donjon.compagnonViewColumn — "active" (défaut, nouvel onglet du même groupe) ou "beside" (vue côte-à-côte).
